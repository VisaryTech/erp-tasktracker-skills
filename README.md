# ERP TaskTracker Skills

Набор скиллов для работы с ERP TaskTracker через Swagger/OpenAPI и CLI-обертки.

## Что внутри

- компактные индексы методов для быстрого выбора нужной команды;
- CLI-точка входа для вызова TaskTracker API;
- Python-клиент, сгенерированный из Swagger-описания.

## Требования

- Python 3.10+
- доступ к ERP TaskTracker API
- настроенные переменные окружения или локальные файлы конфигурации с параметрами подключения и секретами

## Конфигурация

Скиллы читают:

- endpoint API из конфигурации окружения;
- URL для получения токена из конфигурации окружения;
- client id и client secret из переменных окружения или локальных секретов пользователя.

Для локального запуска можно использовать пользовательскую директорию конфигурации:

```text
~/.config/erp/api_base_url
~/.config/erp/token_url
~/.config/erp/client_id
~/.config/erp/client_secret
```

В этих файлах хранятся значения endpoint, token URL и секретов для локальной машины.

## Секреты

Секреты не должны храниться в репозитории.

Рекомендуется передавать их:

- через переменные окружения;
- через локальные пользовательские файлы конфигурации и секретов.

## Назначение

Этот репозиторий предназначен для агентного вызова методов ERP TaskTracker без ручного описания endpoint-ов и без догадок о структуре API.
